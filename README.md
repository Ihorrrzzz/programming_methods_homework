# Спеціальні методи програмування. C++ для астрономії та астрофізики

Репозиторій містить домашні завдання університетського курсу. Кожне незалежне завдання оформлене як окрема консольна програма з власною CMake-ціллю. Графічне завдання з датою реалізоване окремим Qt-застосунком.

## Структура проєкту

```text
.
├── CMakeLists.txt             # усі цілі та параметри складання
├── data/
│   └── sample_text.txt        # короткий приклад для аналізу слів
├── include/astro/
│   ├── date_utils.hpp         # спільні календарні функції
│   └── units.hpp              # сталі та перетворення одиниць
├── src/                       # реалізації окремих завдань
└── tests/
    └── date_utils_test.cpp    # прості перевірки календарних функцій
```

## Вимоги

- компілятор із підтримкою C++17 (Apple Clang, Clang, GCC або MSVC);
- CMake 3.20 або новіший;
- Qt 5 або Qt 6 з компонентом Widgets — лише для завдання 24.

Qt не потрібен для складання консольних програм. Якщо Qt Widgets не знайдено, CMake пропустить тільки ціль `qt_date_app`.

## Складання

Складання виконується поза деревом вихідного коду:

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
```

Для перевірки календарних функцій:

```sh
ctest --test-dir build --output-on-failure
```

Окрему ціль можна зібрати так:

```sh
cmake --build build --target determinant
```

## Запуск програм

Після складання назва виконуваного файла збігається з назвою цілі. Наприклад:

```sh
./build/product
./build/triangle_geometry
./build/determinant
```

У Windows виконувані файли розміщуються у каталозі, який залежить від генератора, наприклад `build/Release/product.exe`.

## Частотний аналіз тексту

Шлях до текстового файла передається першим аргументом:

```sh
./build/word_frequency data/sample_text.txt
```

Якщо аргумент не вказано, програма попросить ввести відносний або абсолютний шлях. Текст може містити звичайні слова, пробіли та розділові знаки. Для латинських слів регістр і розділові знаки не впливають на підрахунок: `apple`, `Apple` та `apple.` вважаються одним словом. Для аналізу повного тексту потрібно покласти власний файл у `data/` або в інший доступний каталог і передати його шлях програмі. У репозиторії навмисно зберігається лише невеликий тестовий текст.

## Qt-застосунок для дати

CMake автоматично шукає Qt 6, а потім Qt 5. Для Homebrew за потреби можна явно передати шлях:

```sh
cmake -S . -B build -DCMAKE_PREFIX_PATH="$(brew --prefix qt)"
cmake --build build --target qt_date_app
```

Запуск на macOS:

```sh
open build/qt_date_app.app
```

На Linux або Windows застосунок запускається як звичайний виконуваний файл `qt_date_app`. Користувач вводить день, місяць і рік; програма перевіряє дату та показує кількість повністю минулих днів від початку року і кількість днів, що залишилися після введеної дати.

## Qt Data Visualization

Qt Data Visualization призначений для апаратно прискореного тривимірного подання даних. Модуль підтримує стовпчикові графіки (`Q3DBars`), діаграми розсіювання (`Q3DScatter`) і поверхні (`Q3DSurface`). У CMake він підключається так:

```cmake
find_package(Qt6 REQUIRED COMPONENTS DataVisualization)
target_link_libraries(my_target PRIVATE Qt6::DataVisualization)
```

Мінімальна програма створює відповідний об'єкт графіка, додає серію даних і викликає `show()`. Докладні приклади наведено в [офіційній документації Qt Data Visualization](https://doc.qt.io/qt-6/qtdatavisualization-index.html). Починаючи з Qt 6.10 модуль позначено застарілим; для нових проєктів Qt рекомендує Qt Graphs, але в цьому репозиторії описано саме навчальний модуль з умови курсу.
